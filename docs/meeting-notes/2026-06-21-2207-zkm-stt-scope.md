**Topic:** Scope `zkm-stt` (`id:dcf8`) — a speech-to-text plugin turning audio sources into timestamped transcript `.md`, with WhatsApp voice messages and YouTube as the driving use cases.

## Surfaced prior art (helferli STT work — reusable)
- whisper.cpp `whisper-server` on Vulkan/Intel Arc is the empirically-verified path (small ≈629 ms); **faster-whisper rejected** (CTranslate2 CPU/CUDA-only, silently falls to CPU on Arc).
- whisper.cpp dropped the OpenAI `/v1/audio/transcriptions` route — its endpoint is `/inference`; need `response_format=verbose_json` to obtain a language field.
- Sample-rate lesson: 8 kHz capture vs 16 kHz model expectation broke accuracy until ffmpeg resampled.
- helferli has no diarisation / no word-level timestamps — greenfield.
- Configurable ASR endpoint URL from day one was the single best reuse lesson.

## Discussion

### Item 1 — Backend & endpoint contract
Archie favoured an OpenAI-compatible default for consistency with `embed.py`/`query.py`. Aria
flagged that whisper.cpp's `whisper-server` exposes `/inference`, **not** `/v1/audio/transcriptions`
— so an OpenAI-only client can't talk to the box Tobias actually runs (`~/src/whisper.cpp`). Riku:
an OpenAI default would be dead-on-arrival for that machine (the helferli bug repeated) and violates
"sensible defaults always"; also a remote `stt_endpoint` ships private voice notes off-box, so the
default must be local. Petra invoked N=2: one user, one machine — ship **one** adapter, leave a
config seam, don't build two speculatively. Mira: the model must stay config (`stt_model`); on the
"whisper alternative not yet checked," the DE+Schweizerdeutsch reality rules out English-strong
fast models (Parakeet/Canary) — Whisper-family or Voxtral are the realistic alternates.

### Item 2 — Integration shape
Three shapes mapped to existing patterns: (A) amender scoped to `created` voice-paths; (B) standalone
`zkm convert stt` over `inbox/stt/`; (C) `zkm fetch youtube`. Aria: WhatsApp notes are already
in-store (amender territory) while YouTube needs the deferred `zkm fetch` core role — serving both in
v1 means a converter + amender + touching deferred core. Petra: v1 = (B) standalone, the source-agnostic
baseline that works today; (A) is the natural v2 once zkm-whatsapp ships (W-pilot still gated). Riku
warned that (B) alone does NOT transcribe the WhatsApp day-file stubs — so the meeting must say "v1 =
generic audio→transcript, WhatsApp integration later" out loud. Archie: factor the work into
`transcribe(audio_path, config) -> Transcript` so the future amender reuses it (seam, not rework).
Mira: (B) also gives a clean bench for the requested multi-model voting eval.

**Factual answers surfaced (read from code/docs):**
- *What zkm-whatsapp does with voice notes today:* `_handle_media` (`plugins/zkm-whatsapp/convert.py:560`)
  copies the audio into CAS (`chat/whatsapp/<tid>/originals/_objects/…`, a copy not a move) + canonical
  inbox symlink + `.origin.json`. The body line is **already** `[media: audio/ogg; codecs=opus → <cas_rel>]
  <!-- key_id: … -->` (`convert.py:282`) — it already references the CAS object AND key_id; the manifest
  persists `media:{mime,sha256}` (w6f). So a future zkm-stt amender needs **no** zkm-whatsapp change to
  find voice notes — enumerate `mime: audio/*`, resolve CAS by sha256, transcribe, augment the line.
- *Are processed inbox files removed?* **No — never automatically.** `docs/plugin-spec.md:316`: inbox is a
  shared zone, a plugin must not process/rewrite/delete items it didn't produce (ownership via
  `.origin.json` `producers[]`). `docs/object-storage.md:113-114`: removal is explicit + ref-counted only
  via `zkm rm` (decrement producers → drop symlink if last → drop CAS object if unreferenced; dry-run
  default) and `zkm gc`. Processing leaves source files in place.

### Items 3-5 — Caching, output shape, scope cuts
Aria: cache keys on `(audio sha256, backend, model, model_version)` — the same shape as the zkm-ner
extraction-cache; store as a CAS-keyed sidecar (`<audio-sha256>.stt.json`); a `model_version` bump busts
the cache. Archie: standalone emits `transcripts/<name>.md` with `source: stt`, `original:`, `sha256:`,
`stt_model`/`stt_backend`/`language` (detected); body = transcript; **segment-level `[mm:ss]`** default.
Mira: defer word-level (10× artifact); auto-detect language, never pin (weak detector worse than nothing;
Schweizerdeutsch hallucinates Arabic/Hindi at small tier); `verbose_json` required for the language field.
Petra named the v1 cuts. Riku added two musts: CAS the audio original (don't leave it only in `inbox/stt/`),
and fail gracefully per-file when the backend is unreachable (skip + warn, like embed's fallback);
minimum evidence = a hermetic mocked-`/inference` test + one real `.opus` smoke test.

## Decisions

- **D1 — Backend/endpoint.** v1 default = **whisper.cpp native `/inference`** request style, **local-only**
  by default. Endpoint URL + model are config (`stt_endpoint`, `stt_model`, secret `stt_api_key` if a remote
  is ever configured). Ship **one** adapter; leave a `stt_api_style: whispercpp | openai` config seam,
  wire the second style only when a second backend is real (N=2). Send `response_format=verbose_json` to get
  the detected language. *Out of scope:* building the OpenAI-compatible adapter now.
- **D2 — Integration shape.** v1 = **standalone `zkm convert stt`** over `inbox/stt/` → `transcripts/<name>.md`.
  Transcription logic factored into `transcribe(audio_path, config) -> Transcript` as the reuse seam.
  *Explicitly out of v1:* WhatsApp inline-integration (the day-file stubs are NOT transcribed in v1) and
  YouTube fetch.
- **D3 — Caching.** Sidecar cache keyed on `(audio sha256, backend, model, model_version)`, mirroring the
  zkm-ner extraction-cache; `model_version` bump busts it. *Out of scope:* a shared core cache library
  (reuse the existing pattern; promote to shared only at N=2).
- **D4 — Output & timestamps.** `transcripts/<name>.md`, frontmatter `source: stt` + `original:` + `sha256:`
  + `stt_model`/`stt_backend`/`language` (detected, never pinned). **Segment-level `[mm:ss]`** body lines.
  Audio original copied to CAS like every plugin; source file left in `inbox/stt/` (no auto-removal — explicit
  `zkm rm`/`zkm gc` only). *Out of scope:* word-level timestamps.
- **D5 — Resilience.** Per-file graceful skip + warn on backend error; never abort the whole convert.
  Done-check: hermetic test mocking `/inference` + one real `.opus` smoke test against the local server.
- **D6 — Deferrals (named).** WhatsApp amender (v2, gated on zkm-whatsapp shipping); YouTube fetch (gated on
  deferred core `zkm fetch` role); diarisation/speaker labels; word-level timestamps; streaming; multi-model
  voting/agreement eval (separate quality harness); background-noise identification/filtering; sentiment
  analysis. All v2+ roadmap.
